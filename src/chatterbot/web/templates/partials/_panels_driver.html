{# Reorder + collapse driver for /insights panels. Every container marked `data-insights-panels` becomes draggable (SortableJS) with per-panel collapse toggles. State persists to localStorage so the streamer's preferences survive both reloads and SSE-driven swaps of the body. The driver re-applies on `htmx:afterSwap` so the panel container getting replaced (engagement view's 60s body refresh, topics view's hx-trigger="insights:topics from:body" refetch) doesn't reset the streamer's order. SortableJS itself is re-initialised on each swap because the container element is new. localStorage keys: chatterbot.insightsPanelOrder — JSON array of panel ids chatterbot.insightsPanelCollapsed — JSON array of panel ids #}
